iOS 线程同步: @synchronized指令

Jun 2nd, 2015

在之前的一个app中，因为对多线程同步的处理不够好，在某些极端情况下app会发生crash。本文简单介绍一下如何使用@synchronized指令来实现线程安全。苹果文档中对@synchronized指令的说明是：

1.@synchronized指令是Objective-C代码中快捷创建互斥锁的一种方式；

2.@synchronized指令像任何其他的互斥锁一样，防止同一时间不同的线程获取相同的锁。

@synchronized指令的使用很简单，只需要传一个参数即可：

@synchronized(key) { 
    // thread-safe code goes here 
}

一个简单的例子：

定义一个全局的数组变量

NSMutableArray *array;

初始化该数组

array = [NSMutableArray arrayWithCapacity:5];
for (int i = 0; i < 5; i++) {
    [array addObject:[NSString stringWithFormat:@"array-%i", i]];
}

创建更新数组的方法

- (void)updateArray:(NSString *)value {

    for (int j = 0; j < array.count; j++) {

        NSString *currentObject = [array objectAtIndex:j];
        [array replaceObjectAtIndex:j withObject:[currentObject stringByAppendingFormat:@"-%@", value]];

        NSLog(@"%@", [array objectAtIndex:j]);
    }
}

在不同的线程中调用修改数组的方法

NSString *bla = @"bla";
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self updateArray:bla];
});
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self updateArray:bla];
});

得到如下结果

array-0-bla
array-0-bla
array-1-bla
array-1-bla-bla
array-2-bla-bla
array-2-bla
array-3-bla
array-3-bla-bla
array-4-bla
array-4-bla-bla

显而易见，两个线程在争夺资源，当两个线程彼此作用时可能会导致app的crash。下面，修改一下更新数组的方法：

- (void)updateArray:(NSString *)value {

    @synchronized (value) {
        for (int j = 0; j < array.count; j++) {

            NSString *currentObject = [array objectAtIndex:j];
            [array replaceObjectAtIndex:j withObject:[currentObject stringByAppendingFormat:@"-%@", value]];

            NSLog(@"%@", [array objectAtIndex:j]);
        }
    }
}

看看结果：

array-0-bla
array-1-bla
array-2-bla
array-3-bla
array-4-bla
array-0-bla-bla
array-1-bla-bla
array-2-bla-bla
array-3-bla-bla
array-4-bla-bla

这下感觉醒目多了～原因就是@synchronized指令的代码块暂停了其中一个线程，允许另一个线程访问代码块，一旦另一个线程完成工作，就取消暂停。

注意了，如果你使用相同的对象作为@synchronized的key，那么线程将会被暂停（正如刚才所见）。假如使用不同的对象作为@synchronized的key：

NSString *bla = @"bla";
NSString *ola = @"ola";
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self updateArray:bla];
});
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    [self updateArray:ola];
});

再看看结果：

array-0-bla
array-0-ola
array-1-bla
array-1-bla-ola
array-2-bla
array-2-bla-ola
array-3-bla
array-3-bla-ola
array-4-bla
array-4-bla-ola

像这种出现多个key的情况，使用

@synchronized (array) { 
    ...

替换

@synchronized (value) { 
    ...

这下妥妥了：

array-0-bla
array-1-bla
array-2-bla
array-3-bla
array-4-bla
array-0-bla-ola
array-1-bla-ola
array-2-bla-ola
array-3-bla-ola
array-4-bla-ola
